# SEO audit

## Overview

Audit the website for basic SEO issues before making any code changes.

## Requirements

- Review metadata: title, description, canonical, Open Graph, Twitter Card.
- Review heading hierarchy.
- Review image alt text and lazy loading.
- Review semantic HTML landmarks.
- Review robots.txt and sitemap.xml if present.
- Review structured data if present.
- Identify SEO issues, explain their impact, and propose fixes.
- Do not modify any files during this audit.

## Acceptance criteria

- Provide a clear SEO audit report.
- List issues by priority: high, medium, low.
- For each issue, include:
  - the problem;
  - why it matters;
  - recommended fix;
  - affected file(s).
- No code changes are made.
