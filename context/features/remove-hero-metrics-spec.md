# Remove hero-metrics, translate and enlarge hero-checks

## Overview

The hero section had two lists: `<ul class="hero-checks">` (4 checkmark items: Google Maps, SEO de bază, Analytics, Design responsive — hardcoded Romanian-only text, no translation) and `<ul class="hero-metrics">` (4 `.metric-card` items showing a stat number + translated label, e.g. "60+ proiecte livrate").

`hero-metrics` and its 4 stat cards were removed entirely. `hero-checks` was kept, but its text is now translatable, and its text + checkmark icon are slightly larger.

## Requirements

### Remove hero-metrics

- Delete `<ul class="hero-metrics">` from `index.html`, including all 4 `.metric-card` items.
- Remove the now-unused `.hero-metrics`, `.metric-card`, `.metric-value`, `.metric-label` rules from `style.css`, and the `.hero-metrics { grid-template-columns: repeat(4, 1fr); }` override in the `≥640px` media query.
- Remove the now-unused `hero.metric1`–`hero.metric4` keys from both `ro` and `en` in `js/translations.js` (only consumer was the deleted `.metric-label` spans).
- No JS render logic changes needed — metric labels were static markup driven by `data-i18n`, not generated by `js/render.js`.

### Translate hero-checks

- Wrap each check item's text in a `<span data-i18n="hero.checkN">` (N = 1–4) instead of bare text nodes, following the same pattern used elsewhere (e.g. `hero.badge`).
- Add `hero.check1`–`hero.check4` to both `ro` and `en` in `js/translations.js`:
  - `hero.check1`: ro "Google Maps" / en "Google Maps"
  - `hero.check2`: ro "SEO de bază" / en "Basic SEO"
  - `hero.check3`: ro "Analytics" / en "Analytics"
  - `hero.check4`: ro "Design responsive" / en "Responsive design"

### Enlarge hero-checks

- Bump `.hero-checks` `font-size` from `0.875rem` to `1rem`.
- Add `.hero-checks .check-icon { width: 20px; height: 20px; }` to size up the checkmark icon only within the hero section (the shared `.check-icon` class — and its 16×16 size — is left untouched for the pricing section's feature list).
